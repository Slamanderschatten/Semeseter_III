class  Manager
 Enthält alle Methoden, in denen übergeordnete/unabhängige Entities/Objekte angelegt werden. Hierbei ist zu bedenken, dass bei der konsequenten Verwendung von smart pointern Objekte, die dynamisch via smart pointern angelegt werden, während der Konstruktion "this" nicht an andere Objekte weitergeben können. Grund hierfür ist, dass das Objekt ggf. in einem nicht vollständig initialisierten Zustand von anderen verwendet wird, was zu undefiniertem Verhalten führen kann. So könnte also ein Renderable in seinem eigenen Konstruktor sich nicht selbst in das ECS eintragen, sondern die Instanz, die das Renderable angelegt hat, müsste das separat tun. Da das unelegant wäre, gibt es die Manager-Klasse, die diese Aufgaben gesammelt übernimmt. Mehr ...
